Installation

Install with npm install @kevin-wynn/nordiccss@0.2.13

If you get 404 errors that the package is not found. Since it's hosted with githubs package repository you might need to add a .npmrc file to your project directory. And add this field to it:

registry=https://npm.pkg.github.com/kevin-wynn

Reference in your application's html like:

<link rel="stylesheet" href="nordic.css" />

You can also reference it via your node modules from your react or vue app

@import 'node_modules/@kevin-wynn/nordiccss/dist/nordic.css';

The Grid

The grid is built upon a very simple basis of a 12 column grid. Using the style of grid-* you can replace each grid with 1-12 for your column size. Each grid has no padding and no margins but comes completely mobile friendly. We will discuss padding and margins in the next section.

Note: Grids must be placed within a grid-container and nested within a row div. For example: 

            <div class="grid-container">
  <div class="row">
    <div class="grid-6"></div>
  </div>
</div>

For responsiveness we have 3 alternative classes you can place alongside the grid size. grid-responsive-small | medium | large. These classes basically transform the grids into a grid-3, grid-6, and grid-12 respectively.

Grid Helpers

The grid helpers are a few extra classes you can apply to give a more customizable feel.

Alignment

You can use the class align-center to center content inside a flex element.

You can use the class align-left to align left content inside a flex element.

You can use the class align-right to align right inside a flex element.

Column and Row

column and row allow you to set the flex direction for an element.

Others

  • grid-fluid - Makes the grid 100% width
  • flex-inline - Gives the display flex-inline to make the div the side of its content
  • fixed - Adds the fixed property to an element
  • no-gutter - Removes the gutter spacing between grid columns

Responsive Helpers

  • center-responsive - Centers an element horizontally in a grid on mobile devices.
  • hide-responsive - Hides the element from responsive devices.
  • show-responsive - Shows the element only on responsive devices.

Margin and Padding

Margins and padding are set up to give a very white space oriented feel. Open and airy is ideal here in order to dray more attention to the actual content. There are 3 different sizes for each margin and padding; small, medium, and large.

Note: You can also define specific margin and padding directions. For example: margin-bottom-small or margin-right-medium, etc...

Note: You can also remove padding and margin using the calsses: margin-left-none or margin-none and similarly with padding.

Forms and Buttons

Nordic CSS has a few stylings for forms and buttons, utilizing some of the colors available for button classes. The forms utlize a few classes to define full width inputs or inline inputs. There are a few pre-defined classes for buttons such as submit and cancel. However you can define any of the available colors using the bg-color-color#. You can easily add a drop shadow using the utility class box-shadow 

            <div class="button-container">
  <div class="button green2">
    Submit
  </div>
</div>

And an example of a form.

            <form class="form-container">
  <p class="text-grey3">Welcome to Nordic CSS. This is an example form.</p>
  <div class="input-group margin-bottom-small">
    <div class="input">
      <label for="firstName">First Name</label>
      <input name="firstName" type="text" />
    </div>
    <div class="input error">
      <label for="lastName">Last Name</label>
      <span class="text-small text-red2">Error Message</span>
      <input name="lastName" type="text" />
    </div>
  </div>
  <div class="input">
    <label for="email">Email</label>
    <input name="email" type="text" />
  </div>
  <div class="button-container">
    <div class="button-group">
      <button class="button submit">Submit</button>
      <button class="button cancel">Cancel</button>
    </div>
  </div>
</form>
